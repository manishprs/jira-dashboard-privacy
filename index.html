Privacy Policy — Jira Executive Dashboard

Last updated: June 2026

Overview

Jira Executive Dashboard ("the Extension") is a Chrome browser extension that provides an executive-grade dashboard view of your Jira boards. This privacy policy explains what data the Extension accesses, how it is used, and what it does not do.

Data we access

The Extension accesses the following data from your Jira instance:

  • Board information — board names, IDs, and types
  • Sprint information — sprint names, start dates, end dates, and status
  • Issue information — issue keys, summaries, statuses, priorities, assignees, created dates, and due dates
  • User information — assignee display names, email addresses, and avatar images

This data is fetched directly from your Jira instance using the Jira REST API, using your existing authenticated browser session.

How data is used

All data accessed by the Extension is used exclusively to render the dashboard panels within your browser. Specifically:

  • Issue data is used to populate the Executive Dashboard, Board View, and People View
  • Assignee data is used to build the Workload Chart and People View
  • Sprint data is used to display sprint progress and health indicators

The Extension does not:

  • Store any Jira data on external servers
  • Transmit any data to any third-party service
  • Log or track your Jira usage
  • Collect any personal information beyond what is displayed in the dashboard
  • Use your data for advertising or analytics

Data storage

The Extension stores only the following in your browser's local storage (chrome.storage.sync):

  • Your list of featured board IDs (board numbers you have selected to appear on the Executive Dashboard)

This preference data:

  • Never leaves your browser except to sync across your own Chrome instances via your Google account (standard Chrome sync)
  • Does not include any Jira issue content, assignee details, or personal information
  • Can be cleared at any time by removing the Extension

Authentication

The Extension uses your existing Jira browser session to make API calls. It does not ask for, store, or transmit your Jira password or API token. All API requests are made directly from your browser to your Jira instance (e.g. https://yourcompany.atlassian.net) using the same authentication cookies that Jira already uses when you browse it normally.

Third-party services

The Extension does not integrate with or send data to any third-party services. There is no backend server associated with this Extension. All processing happens locally in your browser.

Permissions

The Extension requests the following Chrome permissions:

PermissionReason
storageTo save your featured board preferences locally
activeTabTo detect the current Jira page and inject the dashboard panel
Host permission for *.atlassian.net and *.jira.comTo make Jira API calls using your existing session

No other permissions are requested or used.

Children's privacy

This Extension is intended for professional use by engineering managers, CTOs, and product leaders. It is not directed at children under the age of 13. We do not knowingly collect information from children.

Changes to this policy

If this privacy policy changes in a future version of the Extension, the updated date at the top of this page will be revised. Significant changes will be noted in the Extension's Chrome Web Store update notes.
